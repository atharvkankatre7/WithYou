# Privacy Policy for ContentSync

**Last Updated: November 27, 2025**

## Introduction

ContentSync ("we", "our", or "us") operates a mobile application that enables users to synchronize video playback across devices. This Privacy Policy explains how we collect, use, and protect your information.

## Information We Collect

### 1. User Account Information
- User ID (generated by Firebase Authentication)
- Authentication method (anonymous, email, or phone number if provided)
- Account creation date

### 2. File Metadata Only
We **DO NOT** upload, store, or have access to your video files. We only collect:
- SHA-256 hash of your video file (for verification)
- File duration (in milliseconds)
- File size (in bytes)
- Video codec information (format, resolution)

### 3. Room Metadata
- Room ID (randomly generated 6-8 character string)
- Room creation and expiration timestamps
- List of participants (user IDs only)
- Optional room passcode (encrypted)

### 4. Usage Data
- Playback events (play, pause, seek - timestamps only)
- Connection logs (for debugging)
- Chat messages and reactions (temporary, not persisted long-term)

### 5. Technical Information
- IP address (for connection purposes)
- Device type and OS version
- App version

## What We DO NOT Collect

- ❌ Your video files are **never uploaded**
- ❌ Video content is **never accessed** by our servers
- ❌ Personal information beyond authentication
- ❌ Location data
- ❌ Device contacts or gallery access beyond file selection

## How We Use Your Information

1. **Synchronization**: To match files between users and coordinate playback
2. **Authentication**: To verify user identity and secure room access
3. **Service Improvement**: To debug issues and improve performance
4. **Security**: To prevent abuse and ensure service integrity

## Data Storage and Retention

- **Room Metadata**: Automatically deleted after 7 days or when room is closed
- **User Accounts**: Retained while account is active; deleted upon request
- **Event Logs**: Retained for up to 30 days for debugging purposes
- **Chat Messages**: Not persisted; only relayed in real-time

## Data Sharing

We **DO NOT** sell, rent, or share your data with third parties, except:

1. **Firebase (Google)**: For authentication services
2. **Supabase**: For database hosting (encrypted)
3. **Cloud Infrastructure**: Server hosting (Railway/Render)
4. **Legal Compliance**: If required by law

All third-party services are GDPR and CCPA compliant.

## Your Rights

You have the right to:

- **Access**: Request a copy of your data

- **Correct**: Update incorrect information

## Technical Details (For Transparency)

### Data Flow

1. User selects video → App computes SHA-256 hash locally
2. Hash sent to server (not the video file)
3. Server stores hash temporarily for room duration
4. Playback events sent via WebSocket (position + timestamp only)
5. Server relays events between participants
6. All data deleted when room closes or after 7 days

### Third-Party Services

- **Firebase Authentication**: [Firebase Privacy Policy](https://firebase.google.com/support/privacy)
- **Supabase Database**: [Supabase Privacy Policy](https://supabase.com/privacy)
- **Railway/Render Hosting**: [Railway Privacy](https://railway.app/legal/privacy) / [Render Privacy](https://render.com/privacy)

### Data Encryption

- In Transit: TLS 1.3 (HTTPS/WSS)
- At Rest: AES-256 encryption (database)
- File Hashes: SHA-256 (irreversible)
